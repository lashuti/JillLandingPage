JILL - AI Agile Companion Documentation

Overview

JILL is an AI-powered tool designed to streamline Agile workflows by automatically creating detailed tasks from meeting transcriptions. It helps teams save time by automating task creation, assigning them to the right team members, and ensuring they include all necessary acceptance criteria. With JILL, your Agile processes become more efficient, with less manual effort and fewer errors.

Key Features

1. Automated Task Creation

JILL automatically generates tasks from meeting transcriptions, saving your team valuable time and reducing the risk of missed action items.

Each task includes clear summaries and detailed acceptance criteria, created using an advanced AI model.

2. Smart Task Assignment

Tasks are automatically assigned to the right team members, based on their roles and workload.

This feature ensures that no task is overlooked and each team member knows exactly what they're responsible for.

3. Simple-to-Use Interface

JILL features an intuitive, easy-to-use interface where you can simply upload meeting transcription files (e.g., Zoom, Teams) and select the corresponding project.

The system takes care of the rest, generating tasks and assigning them in seconds.

4. Time Savings

What once took 20 minutes to manually create and assign tasks now takes only 2 minutes for review, making your team more efficient and allowing them to focus on delivering results.

FAQs

Q: How do I integrate JILL with Jira?

A: Simply install JILL from the Atlassian Marketplace and follow the installation prompts to connect it to your Jira account.

Q: What meeting transcription formats are supported?

A: JILL supports .vtt files, which are the default for transcriptions and can be downloaded from all popular meeting platforms like Zoom and Microsoft Teams.

Q: How do tasks get assigned to team members?

A: Using an intelligent AI model, JILL determines assignee for the task, then searches your Jira team members and matches them using the names from the meeting's transcription, if there is no match, task is left without an assignee
